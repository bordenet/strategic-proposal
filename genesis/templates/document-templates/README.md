# Genesis Document Templates

This directory contains document template files that define the structure of generated documents.

## Purpose

Document templates are used by the workflow JavaScript to:
1. Define the structure of generated documents
2. Map form fields to document sections
3. Provide formatting and styling

## How to Use

### 1. Create Your Document Template

Create a markdown file that defines your document structure:

```markdown
# {title}

## 1. Executive Summary
{executiveSummary}

## 2. Problem Statement
{problemStatement}

## 3. Proposed Solution
{proposedSolution}

{... more sections ...}
```

### 2. Reference in Your Workflow

In your `js/workflow.js`, load and use the template:

```javascript
async function loadTemplate() {
    const response = await fetch('templates/my-document-template.md');
    return await response.text();
}

function fillTemplate(template, data) {
    let result = template;
    for (const [key, value] of Object.entries(data)) {
        result = result.replace(new RegExp(`{${key}}`, 'g'), value);
    }
    return result;
}
```

### 3. Map Form Fields to Template Variables

In your form, use field IDs that match template variables:

```html
<input type="text" id="title" placeholder="Document Title">
<textarea id="executiveSummary" placeholder="Executive Summary"></textarea>
<textarea id="problemStatement" placeholder="Problem Statement"></textarea>
```

### Complex Template with Sections

See product-requirements-assistant for a complex example with multiple sections, subsections, and conditional content.

## Tips

1. **Use Descriptive Variable Names**: `{problemStatement}` is better than `{field1}`
2. **Match Form Field IDs**: Template variables should match form field IDs exactly
3. **Include Formatting**: Add markdown formatting (headings, lists, etc.) in the template
4. **Test Thoroughly**: Generate documents to verify template works as expected
